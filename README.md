# 三省六部 · Noble Edition

基于 [edict](https://github.com/cft0808/edict) 改造的个人 AI Agent 协作系统。

## 架构

```
皇上（你）—— 批奏折、做决策
    ↑
    │ 每日结报 / 奏折
    │
┌──────────────────────────────────────────────┐
│              太子（分拣层）                    │
│  闲聊自动回复 · 旨意建任务 → 中书省            │
└──────────────────────────────────────────────┘
    ↓ 接旨
┌──────────────────────────────────────────────┐
│              中书省（规划层）                  │
│  接旨 → 规划方案 → 拆解子任务 → 门下省审议   │
└──────────────────────────────────────────────┘
    ↓ 准奏后
┌──────────────────────────────────────────────┐
│              门下省（审核层）                  │
│  审议方案 → 准奏 / 封驳（最多3轮）           │
└──────────────────────────────────────────────┘
    ↓ 准奏
┌──────────────────────────────────────────────┐
│              尚书省（派发层）                  │
│  派发任务 → 协调六部 → 汇总回奏              │
│  每日结报时派发刑部巡查任务                  │
└──────────────────────────────────────────────┘
    ↓ 派发
┌──────┬──────┬──────┬──────┐
│户部  │礼部  │兵部  │工部  │
│理财  │生活  │技能  │开发  │
└──────┴──────┴──────┴──────┘
    ↓ 完成
┌──────────────────────────────────────────────┐
│              刑部（每日巡查）                  │
│  尚书省每日结报时派发 → 巡查所有部门产出      │
│  发现问题 → 汇总奏折呈报皇上                  │
└──────────────────────────────────────────────┘
```

## 部门职责

| 部门 | 职责 | 核心技能 |
|------|------|----------|
| **太子** | 消息分拣：闲聊直接回，旨意建任务 | 分拣判断 |
| **中书省** | 接旨→规划→拆解→提交审议→转尚书省 | 规划分析 |
| **门下省** | 审议方案：准奏/封驳 | 质量审核 |
| **尚书省** | 派发→协调→汇总 | 任务调度 |
| **户部** | 理财/股票/基金/量化策略/定时推送 | 行情API、资讯聚合 |
| **礼部** | 生活提醒/出行/Obsidian收藏(mark) | 日历天气API、知识管理 |
| **兵部** | 技能查找/审核/安装/创造/隐私脱敏审查 | 技能管理、隐私审计 |
| **工部** | 项目开发落地 | Claude Code |
| **刑部** | 每日巡查复盘：效率评估/技能缺口发现 | 数据分析 |

## 工作日志机制

1. **尚书省派发任务时** → 自动创建任务日志模板
2. **部门完成任务后** → 填写3字段（完成情况/产出结果/后续建议）
3. **刑部每日巡查** → 系统汇总日报，刑部快速判断是否有异常
4. **无事发生** → 刑部确认"无事发生，奏报皇上"
5. **发现问题** → 刑部深挖原始日志，汇总奏折呈报皇上

## 隐私脱敏

兵部负责所有部门输出的隐私脱敏审查：

| 类型 | 处理方式 |
|------|----------|
| 资金量 | 改为"重仓"/"轻仓" |
| 账户信息 | 删除或脱敏 |
| 身份信息 | 完全删除 |
| API密钥 | 完全删除 |
| 姓名 | 改为"相关人员" |

## 太子的分拣规则

**直接回复（闲聊）— 不建任务：**
- 简短回复：「好」「否」「?」「了解」「收到」
- 内容不足10个字
- 闲聊/问答/信息查询

**转交中书省（旨意）— 建任务：**
- 明确的工作指令：「帮我做XX」「调研XX」
- 以「传旨」「下旨」开头
- 有实质内容（≥10字）+ 动作词 + 具体目标
